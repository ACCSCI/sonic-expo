# Sonic - 需求文档（马斯克五步重构版）

## 核心目标

打造一个极致简洁、流畅的 Bilibili 音乐播放器

---

## 第一步：质疑需求（Make Requirements Less Dumb）

### 原始需求清单

1. ✅ 保存播放进度和播放列表（及下载状态），启动时扫描下载目录
2. ✅ 设置页面显示当前缓存大小及清除缓存按钮（需要2次确认）
3. ✅ 设置页面显示当前下载歌曲大小及删除所有下载歌曲按钮（需要2次确认）
4. ✅ 设置页面增加明暗主题切换
5. ✅ 进度条改成可以拖动的，按住时进度头放大
6. ✅ 全屏播放控件下滑关闭,左滑右滑切换歌曲（相当于上一首下一首）
7. ✅ 底部导航栏美化
8. ✅ 美化所有图标的UI
9. ✅ 更新readme.md

### 质疑分析

- **为什么要2次确认？** → 用户删除操作频率低，1次确认足够
- **为什么要区分缓存和下载？** → 用户不关心技术细节，统一显示"已用空间"即可
- **为什么要美化所有图标？** → 统一替换图标库比逐个美化更高效
- **为什么要扫描下载目录？** → 下载状态应在下载时同步记录，无需重复扫描

## 第二步：删除（Delete）

### 删除项

| 原需求             | 删除原因             | 简化方案           |
| ------------------ | -------------------- | ------------------ |
| 2次确认删除        | 过度保护，增加摩擦   | 1次确认+可撤销提示 |
| 分离缓存/下载显示  | 技术细节干扰用户     | 合并为"存储空间"   |
| 启动时扫描下载目录 | 低效且不必要         | 下载时同步更新状态 |
| 全屏播放手势       | 与迷你播放器手势冲突 | 仅保留全屏必要手势 |
| 单独美化每个图标   | 工作量大收益低       | 统一替换图标库     |

### 保留核心项

- ✅ 播放进度自动保存
- ✅ 播放列表持久化
- ✅ 下载状态实时同步
- ✅ 存储空间管理
- ✅ 主题切换
- ✅ 可拖动进度条
- ✅ 全屏基础手势
- ✅ 导航栏视觉升级

## 第三步：优化（Optimize）

### 3.1 状态管理 - 极简方案

```
存储键：@sonic_state（单文件）
数据结构：{ queue, currentTrack, position, theme, version }
保存策略：防抖5秒 + 关键事件（暂停/切歌/退后台）
```

### 3.2 存储空间显示 - 合并简化

```
设置页面显示：
├─ 存储空间（进度条）
│  ├─ 已下载：XX MB
│  └─ 缓存：XX MB
└─ [清理空间] 按钮（1次确认）
```

### 3.3 手势交互 - 冲突最小化

| 区域                       | 手势   | 动作         |
| -------------------------- | ------ | ------------ |
| 进度条                     | 拖动   | 调整播放位置 |
| 全屏播放器（非进度条区域） | 下滑   | 关闭全屏     |
| 全屏播放器（非进度条区域） | 左右滑 | 切歌         |
| 迷你播放器                 | 点击   | 展开全屏     |

### 3.4 图标系统 - 统一替换

```
方案：使用 Lucide React Native
步骤：
1. 安装 lucide-react-native
2. 全局搜索替换图标组件
3. 统一尺寸：导航栏24px，按钮20px，播放控制48px
```

## 第四步：加速（Accelerate）

### 4.1 并行任务分配

```
第1组（独立）：
├── 状态持久化（播放进度/列表）
└── 主题切换系统

第2组（依赖第1组）：

└── 可拖动进度条

第3组（独立UI）：
├── 图标替换
├── 导航栏美化
└── 全屏手势

第4组（收尾）：
└── README更新
```

### 4.2 开发顺序

1. **MVP（2天）**：状态持久化 + 主题切换
2. **核心功能（3天）**：进度条 + 设置页面 + 存储管理
3. **UI优化（2天）**：图标 + 导航栏 + 手势
4. **文档（1天）**：README + 测试

## 技术要点

### 存储策略

```typescript
// 统一状态存储
const STORAGE_KEY = "@sonic_state_v1";
interface AppState {
  queue: QueuedTrack[];
  currentTrackId: string | null;
  position: number;
  theme: "light" | "dark";
  version: number;
}
```

### 手势区域划分

```
┌─────────────────────────────────┐
│        全屏播放器               │
│  ┌─────────────────────────┐    │
│  │                         │    │
│  │    上下滑：无响应       │    │
│  │    左右滑：切歌         │    │
│  │                         │    │
│  └─────────────────────────┘    │
│  ┌─────────────────────────┐    │
│  │ ○━━━━━━━━━━━●━━━━━━━━○  │    │
│  │   拖动区域：调进度      │    │
│  └─────────────────────────┘    │
│  [ 点击区域：展开/收起 ]        │
└─────────────────────────────────┘
```

### 性能目标

- 状态保存：异步，不阻塞UI
- 主题切换：<100ms
- 手势响应：<50ms
- 内存占用：增量<10MB
