# Sonic - 需求文档（音频后台与切歌）

## 背景

当前使用 `expo-audio` 在 Android 后台播放时，单曲可播完但无法自动切下一首。
原因是后台时 JS 线程被挂起，导致切歌逻辑无法执行。

## 目标

迁移到 `react-native-track-player`，确保 Android 后台自动切歌可用，
并在后续加入通知栏/锁屏控制。

## 约束

- 平台以 Android 为主，使用 EAS Build（本地无 Java 环境）
- 保留现有下载与缓存逻辑（在线播放 + 本地缓存均需可播放）
- 优先完成基础功能，验证成功后再做通知栏/锁屏控制

---

## 阶段一：基础播放（P0）

### 需求

- 使用 `react-native-track-player` 作为唯一播放器
- 支持队列播放与自动切歌（前台/后台）
- 同时支持在线播放与本地缓存文件播放
- 保持现有 UI 行为不变（尽量复用 `PlayerStore` 接口）

### 验收

- 前台播放结束后自动切下一首
- 后台播放结束后自动切下一首
- 本地缓存与在线播放都可正常切歌

---

## 阶段二：媒体控制（P1）

### 需求

- 通知栏媒体控制（播放/暂停/上一首/下一首）
- 锁屏控制与耳机按键
- 播放元数据正确显示（标题/作者/封面）

### 验收

- 通知栏与锁屏控制响应正确
- 背景切歌稳定，状态同步不丢失

---

## 实施顺序

1. 接入 Track Player 并替换 `PlayerStore` 底层实现
2. 验证阶段一通过后，再启用通知栏/锁屏控制

---

## 测试说明

当前无自动化测试，采用手动验证：

- 前台/后台播放完整曲目，确认自动切歌
- 本地缓存/在线播放各验证 2 首以上
